

# AGENTS.md

> 本文件用于告诉 Codex / AI 编程助手：在本项目中应该遵守哪些协作规则、编码规则、读取文件规则和测试约束。
> 使用说明：每次在这个项目里开始新任务时，应优先读取本文件，并按这里的规则执行。
> 命令：先读取项目根目录的 AGENTS.md，并严格按里面的规则工作。

## 文件编码与读取规则

1. 本项目中的源码、配置、测试、Markdown 文档、SQL、日志样例等文本文件，默认按 UTF-8 编码处理。
2. 在 PowerShell 中读取任何文本文件时，优先显式使用 UTF-8 编码，例如：
   `Get-Content <路径> -Encoding UTF8`
3. 如果看到中文乱码，先怀疑读取命令、终端输出编码或文件实际编码，不要立刻修改文件内容。
4. 在修改包含中文的文件之前，先确认文件内容是否被正确读取；如果原文件是 UTF-8，编辑后必须保持 UTF-8。
5. 不允许仅因为终端输出看起来乱码，就直接替换、删除或重写中文内容。必须先用明确的 UTF-8 方式重新读取文件。
6. 排查问题时遵循这个顺序：
   编码检查 -> 文件内容确认 -> 代码分析 -> 文件修改

## 代码修改规则

1. 修改应尽量小而准确，不要做无关的大范围改动。
2. 不要回退用户未明确要求撤销的已有改动。
3. 对包含中文注释、中文字符串、中文断言内容的文件，修改前后都要确认编码安全。
4. 注释应简洁、直接、实用，不要写无意义的描述性废话。
5. 如果只是测试问题，优先修测试隔离，不要轻易改动全局配置。

## Spring Boot 测试规则

1. MongoDB 相关测试优先使用 `@DataMongoTest`。
2. 只测试 MongoDB 时，应避免加载 JDBC、DataSource 或其他无关数据库配置。
3. 只有在需要验证整个应用上下文时，才使用 `@SpringBootTest`。
4. 如果测试目标只是某一层或某一种存储，不要让测试依赖另一种数据库是否可用。

## PowerShell 使用规则

1. 查看文件内容时，优先使用：
   `Get-Content <路径> -Encoding UTF8`
2. 如果需要带行号查看文件，先保证读取方式是 UTF-8 安全的，再追加行号输出。
3. 如果终端输出仍然异常，先检查控制台编码，不要直接判定文件损坏。

## 协作目标

1. 先确认上下文，再修改代码，不要凭猜测改动。
2. 优先给出可执行、可验证、对当前项目真实有帮助的修改。
3. 在双电脑开发环境下，尽量避免依赖本地 IDE 私有配置或偶然环境状态。
